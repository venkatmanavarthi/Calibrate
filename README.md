# Calibrate

Tailored resumes and job application automation powered by your own AI keys.

Calibrate is an Electron desktop app that uses your own AI provider keys to generate job-specific resumes from reusable profiles and templates. It can also scan tracked companies, score jobs, generate tailored resumes, and apply through your real Chrome browser.

## Features

- Generate tailored resumes by combining a profile (your experience) with a template (formatting/style) against a job description
- Supports multiple AI providers: Anthropic, OpenAI, Google Gemini, Groq, DeepSeek, and LM Studio (local)
- Export to PDF natively from Electron
- API keys stored encrypted via Electron `safeStorage` — never included in profile or template exports
- Markdown editor with live preview for profiles and templates
- Track companies on Greenhouse, Lever, and Ashby; refresh jobs into a local SQLite cache
- Build pipelines that score jobs, generate resumes, and optionally auto-apply to matching roles
- Fill and submit applications in real Chrome through the bundled Calibrate Chrome extension
- Connect Gmail with read-only OAuth to find verification codes or magic links during job-site account creation/login

## Tech Stack

- **Electron** + **electron-vite** — desktop shell and build tooling
- **React 18** + **TypeScript** — renderer UI
- **Tailwind CSS** + **shadcn/ui** (Radix UI) — styling and components
- **Zustand** — state management
- **better-sqlite3** — local jobs, pipelines, application history, and apply-run audit storage
- **Bun** — package manager and script runner

## Getting Started

### Prerequisites

- [Bun](https://bun.sh) installed
- Node.js 20+ (for Electron compatibility)

### Install

```bash
bun install
```

### Run in development

```bash
bun run dev
```

### Build

```bash
bun run build
```

### Typecheck

```bash
bun run typecheck
```

### Package for distribution

```bash
bun run package:mac    # macOS
bun run package:win    # Windows
bun run package:linux  # Linux
```

## Chrome Apply Setup

Calibrate applies through a real Chrome tab using the bundled unpacked extension.

1. Start Calibrate with `bun run dev`.
2. Open Chrome and go to `chrome://extensions`.
3. Enable **Developer mode**.
4. Click **Load unpacked**.
5. Select `extensions/calibrate-chrome`.
6. In Calibrate, open **Settings → Chrome Apply Extension** and click **Check**.

The Electron app starts a local bridge on `127.0.0.1:17654`. The extension polls that bridge for tab commands and reports results back to Calibrate. This is currently a local development bridge, not a packaged native messaging host.

## Gmail Verification Setup

Gmail is optional, but required for fully automatic account creation/login when a job site asks for email verification.

1. Create a Google OAuth desktop client in Google Cloud.
2. In Calibrate, open **Settings → Gmail Verification**.
3. Paste the OAuth client ID and client secret.
4. Click **Save Gmail OAuth Config**.
5. Click **Connect Gmail** and approve the read-only Gmail scope.

Calibrate requests `https://www.googleapis.com/auth/gmail.readonly`. It only searches recent messages for verification codes or magic links. It does not send, delete, label, or modify emails.

## Apply Flow

1. Add companies and refresh jobs.
2. Create or run a pipeline to score jobs.
3. Generate resumes for selected scored jobs.
4. Click submit/apply from the Pipeline page, or enable auto-apply on a pipeline.
5. Calibrate opens Chrome, fills the form, uploads the generated resume PDF, handles account login/creation when possible, submits, and stores an audit record.

Application history and recent Chrome apply runs are shown on the **Applications** page.

Safety limits:

- Calibrate does not solve CAPTCHAs.
- Calibrate does not bypass browser security warnings, paywalls, or non-Gmail MFA.
- Calibrate blocks submission when required fields remain empty.
- Site credentials generated by Calibrate are unique per origin and stored locally with `safeStorage` when available.

## Project Structure

```
electron/
  main/
    ai/            # AI provider integrations (Anthropic, OpenAI-compatible, Gemini)
    chrome-apply/  # Chrome extension bridge and apply orchestration
    session-auth/  # ATS session login helpers
    gmail/         # Gmail OAuth and verification lookup
    ipc/           # IPC handlers exposed to renderer
    jobs/          # ATS job discovery adapters
    pdf/           # PDF export
    security/      # safeStorage key management
    storage/       # Profiles, templates, jobs, pipelines, applications
extensions/
  calibrate-chrome/ # Unpacked Chrome extension for real-browser apply automation
src/
  components/  # Shared UI components
  pages/       # Dashboard, generator, jobs, pipelines, applications, settings
  stores/      # Zustand stores
  types/       # Shared TypeScript types
```

## Notes

- Vite is pinned to `^5.4.x`. `electron-vite@2.3.0` uses `splitVendorChunk` which was removed in Vite 6+. Do not upgrade Vite without also upgrading `electron-vite`.
- The main process builds as CJS via electron-vite's SSR build. Do not add `"type": "module"` to `package.json`.
- Chrome apply currently uses a local HTTP bridge for development. A packaged native messaging host is the right distribution path before shipping this broadly.
- Gmail read-only access may require Google OAuth app verification before public distribution.
